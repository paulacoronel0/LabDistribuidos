con RMI

Interfaces Compartidas (Deben estar disponibles para todos):

ServicioPrediccion.java (Contiene prediccion()).

ServicioCentral.java (Contiene un método como obtenerConsultaCompleta(String signo, String fecha)).

Servidores Backend:

ServidorH.java (Implementa ServicioPrediccion, registra su objeto en RMI).

ServidorP.java (Implementa ServicioPrediccion, registra su objeto en RMI).


Servidor Intermediario:

ServidorCentral.java (Implementa ServicioCentral. En su método, busca los objetos remotos de H y P, hace las llamadas y concatena los resultados).

Cliente:

Cliente.java (Solo busca el objeto remoto del ServidorCentral en el registro y llama al método)



ARQUITECTURA:

Cliente --> ServicioCentral (Interfaz) -->  ServicioPrediccion (Interfaz)
                    |
                -------                                   |
                   |                                      |
ServidorCentral -->      ServicioCentralImp     -------------------------
                                                |                    |
                                                ServidorP           ServidorH



Tener en cuenta:

- Calculadora calc = (Calculadora) Naming.lookup("//localhost:5000/CalculadoraImp"); 
Esto hace un lookup (búsqueda) en el registro RMI.

"//localhost:5000/CalculadoraImp":
localhost → máquina donde corre el servidor
5000 → puerto del rmiregistry
CalculadoraImp → nombre con el que se registró el objeto remoto

Resultado:

Te devuelve un stub (una referencia remota)
Lo casteás a Calculadora




System.setProperty("java.rmi.server.hostname","localhost");

Esto define qué IP/host va a devolver el servidor RMI al cliente

Importante porque:

El cliente recibe una referencia remota
Esa referencia incluye una dirección para volver a conectarse

Ejemplo:

Si estás en red → debería ser:
"192.168.x.x" o IP real del servidor
Si dejás mal esto:
el cliente no puede invocar métodos (error típico)


Naming.rebind("rmi://localhost:" + args[0] + "/CalculadoraImp", calc);
Registrar el objeto remoto

rebind:
Asocia el nombre "CalculadoraImp" con el objeto calc
Si ya existía → lo reemplaza
"localhost:" + args[0]:
Usa el puerto pasado por parámetro
Ej: args[0] = 5000

Resultado:

El objeto queda disponible para clientes
Los clientes pueden hacer lookup con ese nombre
